# CRM Dashboard System (Key Business Engine)

Received: 2026-06-30

The 3-tier dashboard system is the core business engine of Kimure. Each tier
serves a different user role and has its own routes, permissions, and features.

## 3-Level Dashboard Architecture

### A. Individual User Dashboard

For regular users searching for properties. Default role on signup.

**Features:**
- AI property recommendations
- Financial insights
- Saved searches
- AI reports history
- "Connect with agent" CTA

### B. Partner / Business Dashboard (Revenue Engine)

For agents, brokers, and lenders. **This is where Kimure makes money** — partners
pay for qualified leads generated by user intent on the platform.

**Features:**
- Lead inbox (qualified users)
- User intent data (budget, AI reports viewed)
- CRM pipeline stages:
  - New lead
  - Contacted
  - Negotiation
  - Closed
- AI-powered suggestions:
  - "Best time to contact"
  - "Top matching properties"

### C. Admin Dashboard (Super Admin / Support)

Platform staff only. Not self-signup — admins are promoted manually.

**Features:**
- User management
- Agent / broker approvals
- AI usage monitoring
- Revenue tracking (CPL / subscriptions)
- Support ticket system

## Related Database Tables

- `profiles.role` — enum: `individual` · `partner` · `admin` · `support`
- `partners` — business account data linked to a profile
- `leads` — CRM lead records with status pipeline
- `saved_properties` — saved listings per user
- `ai_reports` — user-facing AI report history
- `audit_logs` — admin action trail
